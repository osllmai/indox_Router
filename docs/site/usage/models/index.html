    Model Information & Parameters¶

    IndoxRouter provides access to various AI models from multiple providers. This guide covers model selection, parameters, and capabilities.

    Available Providers & Models¶

    OpenAI Models¶

    • Chat Models: gpt-4o, gpt-4o-mini, gpt-3.5-turbo
    • Text Completion: gpt-3.5-turbo-instruct
    • Embeddings: text-embedding-3-small, text-embedding-3-large, text-embedding-ada-002
    • Image Generation: dall-e-3, dall-e-2

    Anthropic Models¶

    • Chat Models: claude-3-opus-20240229, claude-3-sonnet-20240229, claude-3-haiku-20240307
    • Capabilities: Text completions only (no image generation)

    Google Models¶

    • Chat Models: gemini-1.5-pro, gemini-1.5-flash, gemini-pro
    • Capabilities: Text completions, embeddings

    Mistral Models¶

    • Chat Models: mistral-large-latest, mistral-medium-latest, mistral-small-latest
    • Capabilities: Text completions only

    DeepSeek Models¶

    • Chat Models: deepseek-chat, deepseek-coder
    • Capabilities: Text completions only

    XAI Models¶

    • Chat Models: grok-beta
    • Capabilities: Text completions, image generation

    Qwen Models¶

    • Chat Models: qwen-turbo, qwen-plus, qwen-max
    • Capabilities: Text completions, embeddings, image generation

    Model Parameters¶

    Universal Parameters¶

    These parameters are supported across most providers:

    client.chat(
    messages=[...],
    model="openai/gpt-4o",
    # Universal parameters
    temperature=0.7,        # Controls randomness (0.0-2.0)
    max_tokens=1000,        # Maximum tokens to generate
    top_p=0.9,             # Nucleus sampling parameter
    frequency_penalty=0.0,  # Reduces repetition (-2.0 to 2.0)
    presence_penalty=0.0,   # Encourages new topics (-2.0 to 2.0)
    stream=False           # Enable streaming responses
)

    Parameter Details¶

    Temperature (0.0 - 2.0)¶

    Controls the randomness of the output:

    • 0.0: Deterministic, always selects the most likely token
    • 0.7: Balanced creativity and coherence (recommended for most use cases)
    • 1.0: Standard randomness
    • 2.0: Highly creative but potentially incoherent

    Max Tokens¶

    Maximum number of tokens to generate in the response:

    • Varies by model (see model specifications below)
    • Includes both input and output tokens for some providers
    • Set based on your use case and cost considerations

    Top P (0.0 - 1.0)¶

    Nucleus sampling parameter:

    • Controls the cumulative probability of token choices
    • 0.1: Very focused, only considers top 10% of likely tokens
    • 0.9: More diverse, considers top 90% of likely tokens
    • Alternative to temperature for controlling randomness

    Frequency Penalty (-2.0 to 2.0)¶

    Reduces repetition of tokens based on their frequency:

    • Positive values discourage repetition
    • Negative values encourage repetition
    • 0.0: No penalty (default)

    Presence Penalty (-2.0 to 2.0)¶

    Encourages discussing new topics:

    • Positive values discourage repeating any token that has appeared
    • Negative values encourage repeating tokens
    • 0.0: No penalty (default)

    # Check model capabilities
models = client.models()

for provider in models:
    print(f"Provider: {provider['name']}")

    # Text completion models
    if 'text_completions' in provider:
        for model in provider['text_completions']:
            print(f"  Chat Model: {model['modelName']}")
            if 'specs' in model:
                print(f"    Context: {model['specs'].get('context_window', 'Unknown')}")

    if 'embeddings' in provider:

            print(f"  Embedding Model: {model['modelName']}")
            if 'specs' in model:
                print(f"    Dimensions: {model['specs'].get('dimensions', 'Unknown')}")

    usage = response['usage']
    print(f"Prompt tokens: {usage['tokens_prompt']}")
    print(f"Completion tokens: {usage['tokens_completion']}")
    print(f"Total tokens: {usage['tokens_total']}")

    Cost-Effective Model Selection¶

    1. Development/Testing: Use gpt-3.5-turbo or gpt-4o-mini
    2. Production (High Volume): Use gpt-4o-mini or claude-3-haiku
    3. Production (High Quality): Use gpt-4o or claude-3-sonnet
    4. Specialized Tasks: Choose provider-specific models

    Dynamic Model Selection¶

    def select_model(task_type, quality_level="balanced"):
    """Select optimal model based on task and quality requirements."""

    models = {
        "chat": {
            "fast": "openai/gpt-3.5-turbo",
            "balanced": "openai/gpt-4o-mini",
            "high_quality": "openai/gpt-4o"
        },
        "analysis": {
            "fast": "anthropic/claude-3-haiku-20240307",
            "balanced": "anthropic/claude-3-sonnet-20240229",
            "high_quality": "anthropic/claude-3-opus-20240229"
        },
        "coding": {
            "fast": "openai/gpt-4o-mini",
            "balanced": "openai/gpt-4o",
            "high_quality": "deepseek/deepseek-coder"
        }
    }

    return models.get(task_type, {}).get(quality_level, "openai/gpt-4o-mini")

# Usage
model = select_model("coding", "high_quality")
response = client.chat(messages=[...], model=model)

    List Available Models¶

    # Get all available models
models = client.models()

# Filter by provider
openai_models = [p for p in models if p['id'] == 'openai'][0]

# Filter by capability
chat_models = []
for provider in models:
    if 'text_completions' in provider:
        for model in provider['text_completions']:
            chat_models.append(f"{provider['id']}/{model['modelName']}")

print("Available chat models:", chat_models)

    Model Validation¶

    def validate_model(provider, model_name, operation_type="chat"):
    """Validate if a model supports the requested operation."""
    try:
        models = client.models()

        # Find provider
        provider_data = next((p for p in models if p['id'] == provider), None)
        if not provider_data:
            return False, f"Provider '{provider}' not found"

        # Check if model exists for operation
        operation_map = {
            "chat": "text_completions",
            "completion": "text_completions",
            "embedding": "embeddings",
            "image": "image_generation"
        }

        model_category = operation_map.get(operation_type)
        if not model_category or model_category not in provider_data:
            return False, f"Operation '{operation_type}' not supported by provider"

        # Check if specific model exists
        models_in_category = provider_data[model_category]
        model_exists = any(m['modelName'] == model_name for m in models_in_category)

        if not model_exists:
            available = [m['modelName'] for m in models_in_category]
            return False, f"Model '{model_name}' not found. Available: {available}"

        return True, "Model is valid"

    except Exception as e:
        return False, f"Validation error: {str(e)}"

# Usage
is_valid, message = validate_model("openai", "gpt-4o", "chat")
print(f"Valid: {is_valid}, Message: {message}")
